# Basic Website CMS

작은 조직을 위한 **웹사이트 관리 도구**입니다. 공개 웹사이트와 관리자 콘솔이 한 쌍으로 이루어져 있어, 관리자 화면에서 게시물과 일정을 등록·수정·삭제하면 그 내용이 공개 웹사이트에 반영됩니다.

> **데모에 관하여** — 이 저장소의 화면에 보이는 기관명(`모오 임상심리연구소`, MOW Institute of Clinical Psychology)은 실존하지 않는 가상의 기관이며, 문구·게시물·일정은 전부 예시입니다. 새 집을 계약하기 전에 둘러보는 모델하우스처럼, 구조를 보여주기 위한 가상의 웹사이트입니다. 실제 콘텐츠는 이 템플릿을 가져다 쓰는 분이 채워 넣게 됩니다.

## 왜 만들었나

작은 조직은 웹사이트를 한번 열어둔 뒤 그것을 꾸준히 돌볼 사람이 없는 경우가 많습니다. 오래된 게시판은 시간이 지나며 보안이 낡고, 외부의 스팸·광고 공격에 노출됩니다. 이를 막으려 임시방편을 겹겹이 덧대다 보면, 정작 정상적인 방문자의 접속까지 막히는 일도 생깁니다.

이 도구는 그 자리를 대신하기 위해 **두 가지를 각각** 지향합니다.

- **안전할 것.** 공개 웹사이트 자체가 공격 표면이 되지 않도록 설계했습니다. (아래 [두 가지 설계 원칙](#두-가지-설계-원칙)에서 실현됩니다.)
- **다루기 쉬울 것.** 개발 지식이 없는 운영자도 부담 없이 다룰 수 있도록, 기능을 꼭 필요한 것들로만 추렸습니다.

관리자는 콘솔에서 게시물과 일정을 **등록·수정·삭제**하고, 방문자는 공개 사이트에서 그것을 **열람하고 상호작용**합니다. 실사용 기능은 거기까지입니다.

## 두 가지 설계 원칙

안전함은 이 두 원칙에서 나옵니다.

**1. 방문자의 웹사이트와 관리자의 웹사이트를 나눕니다.**
공개 사이트(`apps/site`)와 관리자 콘솔(`apps/admin`)은 서로 다른 앱으로 배포됩니다. 방문자가 접속하는 쪽에는 관리 기능도, 데이터베이스로 향하는 통로도 존재하지 않습니다.

**2. 방문자의 상호작용은 자유롭되, 사이트 보안에 위험을 더하지 않습니다.**
방문자는 게시물과 일정을 열람하고 상호작용하지만, 어떤 동작도 공개 사이트가 데이터베이스를 직접 건드리게 만들지 않습니다. 공개 사이트는 관리자 콘솔이 내보내는 읽기 전용 데이터만 바라보고, 그마저 닿지 않으면 미리 구워둔 정적 콘텐츠로 조용히 내려앉습니다.

## 구조

```
apps/
  site/    공개 웹사이트  (Next.js App Router, ISR 60초 캐시)
  admin/   관리자 콘솔    (Next.js + Prisma + PostgreSQL, Google OAuth)
```

콘텐츠는 한 방향으로만 흐릅니다.

```
[관리자 콘솔]  ──(읽기 전용 공개 API)──▶  [공개 사이트]
      │                                        ▲
      └──(저장 시 재검증 웹훅, 공유 시크릿)──────┘
```

- 관리자가 콘텐츠를 저장하면, 관리자 콘솔이 공개 사이트의 재검증 엔드포인트를 호출해 캐시를 즉시 갱신합니다(`x-cms-revalidate-secret` 공유 시크릿, 타이밍 세이프 비교).
- 관리자 API를 연결하지 않으면(예: `SITE_ADMIN_API_URL` 미설정), 공개 사이트는 데이터베이스 없이 정적 콘텐츠만으로도 그대로 동작합니다.

## 페이지 유형 — 하나의 템플릿으로서

공개 사이트의 상단 메뉴는 페이지를 어떻게 구성할 수 있는지 세 가지 유형으로 보여줍니다.

- **단일** — 하위 페이지 없는 한 장짜리 페이지. 기관 소개·인사말 같은 콘텐츠에 어울립니다.
- **중첩** — 하위 페이지를 거느린 메뉴. 여러 제품이나 서비스를 나열하는 데 어울립니다(데모에서는 프로그램 A~D). 상위 개요에는 검색·필터가 있는 게시물 목록을, 각 하위 페이지에는 일정 캘린더와 탭 형식의 게시물을 배치했습니다.
- **외부** — 외부 웹사이트로 나가는 링크(데모에서는 이 저장소로 연결됩니다).

가져다 쓰는 분은 이 유형들을 조합해 자신의 사이트를 구성하면 됩니다.

## 주요 기능

**공개 사이트 (`apps/site`)**
- 게시물 카테고리(공지사항 · 안내 · 자유게시판 · 자료실)와 프로그램 라벨 기반 라우팅
- 여러 종류의 일정을 한눈에 보여주는 캘린더(대한민국 공휴일 표시)
- 검색·필터·페이징이 있는 게시물 목록, 게시물 딥링크(`?notice=`)
- 히어로 스크롤 유도, 스크롤에 따라 서서히 바뀌는 헤더 등 절제된 상호작용
- 첨부파일 다운로드 프록시(허용된 스토리지 경로만 통과)
- ISR 60초 캐시 + 태그 재검증, 보안 헤더(CSP 포함), sitemap/robots 자동 생성

**관리자 콘솔 (`apps/admin`)**
- Google OAuth 로그인 + 하드코딩 허용 목록(허용된 계정 외에는 로그인 자체가 거부됨)
- 리치 텍스트 에디터(TinyMCE)로 게시물 작성, 파일·이미지 첨부(Vercel Blob)
- 프로그램 일정 관리: 기수(run) → 단계(stage) → 세션(session) 구조, 신청 기간·신청 링크, 기수 자동 번호
- 일반 일정(기관 캘린더) 관리
- 휴지통(소프트 삭제 → 복원/영구 삭제), 일괄 작업, 작성자 표시명 관리
- 저장되는 HTML은 쓰기·읽기 양쪽에서 새니타이즈(허용 목록 기반), 업로드는 확장자별 형식 검증
- 관리자 도메인은 검색엔진 비노출(`noindex`) + CSP 등 보안 헤더

## 시작하기

요구 사항: Node.js 20+, PostgreSQL(로컬 또는 Neon 등 매니지드).

```bash
# 1. 의존성 설치
npm --prefix apps/site install
npm --prefix apps/admin install

# 2. 환경변수 설정 (각 앱의 .env.example을 .env.local로 복사해 채우기)
#    admin: DATABASE_URL, NEXTAUTH_SECRET, GOOGLE_OAUTH_CLIENT_ID/SECRET 등
#    site : SITE_ADMIN_API_URL(관리자 공개 API 주소) 등

# 3. 데이터베이스 스키마 반영 + 시드
npm run admin:db:push
npm run admin:seed        # 허용 계정 시드
npm run admin:seed:demo   # 데모 게시물·일정 시드(선택)

# 4. 실행 (관리자 3001, 사이트 3000)
npm run admin:dev
npm run site:dev
```

로컬 개발 모드에서는 관리자 로그인 화면에서 **데모 계정으로 콘솔을 바로 둘러볼 수 있습니다** — Google OAuth 설정 없이도요. 이 데모 로그인은 개발 모드에서만 열리며, 프로덕션 빌드에서는 코드 레벨에서 막힙니다.

실제 운영에서는 Google OAuth만 사용합니다. 로그인 허용 계정은 `apps/admin/lib/site-admin/allowed-users.ts`에서 수정합니다. 여기에 등록된 이메일의 Google 계정만 로그인할 수 있습니다.

## 배포

Vercel 기준입니다.

- 저장소 하나에 프로젝트 두 개를 만들고, 각 프로젝트의 **Root Directory**를 `apps/site`, `apps/admin`으로 지정합니다.
- 공개 사이트에는 `SITE_ADMIN_API_URL`(관리자 공개 API)과 `SITE_REVALIDATE_SECRET`을, 관리자 콘솔에는 같은 시크릿과 `SITE_REVALIDATE_URL`(공개 사이트의 재검증 엔드포인트)을 설정하면, 관리자에서 저장하는 즉시 공개 사이트에 반영됩니다.
- 관리자 API 없이 공개 사이트만 정적으로 띄우려면 `SITE_ADMIN_API_URL`을 비워 두면 됩니다.

## 라이선스와 공개 의도

MIT 라이선스입니다. 자유롭게 가져다 쓰시고, 저작권 표기 한 줄만 남겨 주십시오.

이 템플릿의 어느 부분도 무에서 만들어지지 않았습니다. 잘 만들어진 프레임워크와 도구들 위에서, 이미 존재하는 부품을 골라 조립한 결과물입니다. 무엇을 고르고 어떻게 잇는가에 만든 사람의 판단이 담기지만, 같은 부품이 주어진 이상 누군가는 언젠가 비슷한 구조에 도달했을 것입니다. 다만 먼저 도달해 본 사람으로서, 다음 사람이 같은 시행착오를 반복하지 않도록 그 결과를 정리해 둡니다.
