# Basic Website CMS

작은 조직을 위한 **공개 웹사이트 + 관리자 콘솔** 템플릿입니다.

개발 인력이 상주하지 않는 소규모 기관도, 관리자 화면에서 게시물과 일정을 등록·수정·삭제하는 것만으로 공개 웹사이트를 안전하게 운영할 수 있도록 설계했습니다. 실제 소규모 기관의 웹사이트를 구축·운영하며 다듬은 구조를 누구나 가져다 쓸 수 있는 형태로 정리한 것입니다.

> **데모에 관하여** — 이 저장소의 화면에 보이는 기관명(`모오 임상심리연구소`, MOW Institute of Clinical Psychology)은 실존하지 않는 가상의 기관이며, 문구와 게시물은 전부 예시(플레이스홀더)입니다. 새 집을 계약하기 전에 둘러보는 모델하우스처럼, 구조를 보여주기 위한 가상의 웹사이트입니다. 실제 콘텐츠는 이 템플릿을 가져다 쓰는 분이 채워 넣게 됩니다.

## 설계 원칙

1. **방문자의 웹사이트와 관리자의 웹사이트를 분리합니다.** 공개 사이트(`apps/site`)와 관리자 콘솔(`apps/admin`)은 별개의 앱으로 배포됩니다. 방문자가 접속하는 서버에는 관리 기능이 아예 존재하지 않습니다.
2. **방문자의 상호작용은 자유롭게, 보안 위험은 없게.** 방문자는 게시물과 일정을 열람하고 상호작용하지만, 공개 사이트는 데이터베이스에 직접 닿지 않습니다. 관리자 콘솔이 제공하는 읽기 전용 공개 API만 바라보며, 그마저 실패하면 정적 폴백으로 조용히 내려앉습니다.

## 구조

```
apps/
  site/    공개 웹사이트  (Next.js App Router, ISR 60초 캐시)
  admin/   관리자 콘솔    (Next.js + Prisma + PostgreSQL, Google OAuth)
```

콘텐츠 흐름은 한 방향입니다.

```
[관리자 콘솔]  ──(읽기 전용 공개 API)──▶  [공개 사이트]
      │                                        ▲
      └──(저장 시 재검증 웹훅, 공유 시크릿)──────┘
```

- 관리자가 콘텐츠를 저장하면, 관리자 콘솔이 공개 사이트의 재검증 엔드포인트를 호출해 캐시를 즉시 갱신합니다(`x-cms-revalidate-secret` 공유 시크릿, 타이밍 세이프 비교).
- 공개 사이트의 메뉴는 네 가지 유형을 시연합니다: **기본**(단일 페이지) · **중첩**(하위 페이지를 가진 메뉴) · **게시판** · **외부**(외부 링크).

## 주요 기능

**공개 사이트 (`apps/site`)**
- 게시물 카테고리(공지사항 · 안내 · 자유게시판 · 자료실)와 프로그램 라벨 기반 라우팅
- 프로그램별 상세 페이지: 일정 캘린더(대한민국 공휴일 표시) + 게시물 탭
- 검색·필터·페이징이 있는 게시판, 게시물 딥링크(`?notice=`)
- 첨부파일 다운로드 프록시(허용된 스토리지 경로만 통과)
- ISR 60초 캐시 + 태그 재검증, 보안 헤더, sitemap/robots 자동 생성

**관리자 콘솔 (`apps/admin`)**
- Google OAuth 로그인 + 하드코딩 허용 목록(허용된 계정 외에는 로그인 자체가 거부됨)
- 리치 텍스트 에디터(TinyMCE)로 게시물 작성, 파일·이미지 첨부(Vercel Blob)
- 프로그램 일정 관리: 기수(run) → 단계(stage) → 세션(session) 구조, 신청 기간·신청 링크, 기수 자동 번호
- 일반 일정(기관 캘린더) 관리
- 휴지통(소프트 삭제 → 복원/영구 삭제), 일괄 작업, 작성자 표시명 관리
- 저장되는 HTML은 쓰기·읽기 양쪽에서 새니타이즈(허용 목록 기반)
- 관리자 도메인은 검색엔진 비노출(`noindex`) + CSP 등 보안 헤더

## 시작하기

요구 사항: Node.js 24+, PostgreSQL(로컬 또는 Neon 등 매니지드).

```bash
# 1. 의존성 설치
npm --prefix apps/site install
npm --prefix apps/admin install

# 2. 환경변수 설정 (각 앱의 .env.example을 .env.local로 복사해 채우기)
#    admin: DATABASE_URL, NEXTAUTH_SECRET, GOOGLE_OAUTH_CLIENT_ID/SECRET 등
#    site : SITE_ADMIN_API_URL(관리자 공개 API 주소) 등

# 3. 데이터베이스 스키마 반영 + 시드
npm run admin:db:push
npm run admin:seed        # 허용 계정 시드
npm run admin:seed:demo   # 데모 게시물·일정 시드(선택)

# 4. 실행 (관리자 3001, 사이트 3000)
npm run admin:dev
npm run site:dev
```

관리자 로그인 허용 계정은 `apps/admin/lib/site-admin/allowed-users.ts`에서 수정합니다. 여기에 등록된 이메일의 Google 계정만 로그인할 수 있습니다.

## 배포

- Vercel 기준, 저장소 하나에 프로젝트 두 개를 만듭니다: root directory를 각각 `apps/site`, `apps/admin`으로 지정합니다.
- 공개 사이트에는 `SITE_ADMIN_API_URL`(관리자 공개 API)과 `SITE_REVALIDATE_SECRET`을, 관리자 콘솔에는 같은 시크릿과 `SITE_REVALIDATE_URL`(공개 사이트의 재검증 엔드포인트)을 설정하면 저장 즉시 공개 사이트에 반영됩니다.

## 라이선스와 공개 의도

MIT 라이선스입니다. 자유롭게 가져다 쓰시고, 저작권 표기 한 줄만 남겨 주십시오.

이 템플릿의 어느 부분도 무에서 만들어지지 않았습니다. 잘 만들어진 프레임워크와 도구들 위에서, 이미 존재하는 부품을 골라 조립한 결과물입니다. 무엇을 고르고 어떻게 잇는가에 만든 사람의 판단이 담기지만, 같은 부품이 주어진 이상 누군가는 언젠가 비슷한 구조에 도달했을 것입니다. 다만 먼저 도달해 본 사람으로서, 다음 사람이 같은 시행착오를 반복하지 않도록 그 결과를 정리해 둡니다.
